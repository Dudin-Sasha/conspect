
## Тестирование как часть процесса верификации ПО.

Качество программного обеспечения - совокупность характеристик ПО  (надежность, производительность, удобство и т.д.) 

Верификация ПО - Соответствие строгим требованиям качества (заданным требованиям)

Валидация ПО - Соответствие ожиданиям пользователя


## Жизненный цикл ПО (SDLC)

1) Планирование 
2) Анализ
3) Проектирование
4) Разработка
5) Тестирование (происходит всегда, на всех этапах)
6) Поддержка
*( 7. выход из строя )* 


Тестирование ПО - это процесс оценки ПО для выявления различий между фактическим и ожидаемым поведением.

### Цели тестирования

 - Обнаружение дефектов
 - Оценка качества и степени готовности
 - Снижение рисков связанных с релизом

# Когда проводится тестирование

- *В идеальных условиях* непрерывно
- После любых больших изменений проекта
- Обязательно - перед каждым релизом. 


### Уровни тестирования (4 типа, дальше больше)

![[Pasted image 20260117142538.png]]

Модульное тестирование  - Проверка отдельных модулей (функции классы и т.д.) обычно силами разработчика

xграционное тестирование - Взаимодействие и интерфейсы между модулями или системами. 

Системное тестирование  - проверка системы целиком как единого целого в среде в среде близкой к настоящей (проверка приложения как в жизни нужно ли удобно ли работает лиz)

Приемочное тестирование Проверка заказчиком Соответствие продукта бизнес-требованиям,  проводятся заказчиком или его представителем

### Виды тестирования(классификация по цели или объекту тестирования)

- Функциональное, соответствие функции требованиям
- Регрессионное, проверка что новые изменения не сломали старое

- Нагрузочное Проверка производительности и стабильности под нагрузкой

### Тестовая среда
 -  **DEV** Среда разработки
 - **TEST** Выделенная среда для тестирования
 - **STAGE (PRE-PROD)** Максимально точная копия боевой среды
 - **PROD** Боевая (Продуктивная) среда

### Тестовая документация


- Тест-кейс
- Чек-лист 
- Отчет


# Классификация Тестирования ПО по:

## Целям и задачам

**Функциональное тестирование**

Модульное
Интеграционное
Системное
Приемочное


**Нефункциональное тестирование**

Проверка не связанная с какими-либо функциями
- Производительности: Нагрузочное, стресс-тестирование
- Безопасности: Проверка уязвимостей
- Usability: Тестирование удобства использования
- Совместимости: Работа в разных средах и конфигурациях

## По уровню тестирования

#### Статическое тестирование
Анализ кода без выполнения программы
- Образы кода
- Статический анализ(линтеры, анализаторы)
- Инспекции документации

#### Динамическое тестирование
тестируется с выполнением программы
- все виды функц. и нефункц. тестирования
- тестирование в реальном времени

## По степени автоматизации

 **Ручное тестирование** - выполнение тестов без скриптов
	- Плюсы: Гибкость, Человеческая интуиция
	- Минусы: Время, повторяемость, масштабируемость

**Автоматическое тестирование** - Использование скриптов и инструментов
	- Виды: Unit тесты, API- тесты, UI-автоматизация
	- Плюсы: Скорость, повторяемость, покрытие
	- Минусы: Стоимость, поддержка

**Полуавтоматическое тестирование** - Комбинирование Ручного и Автоматического


## По знанию внутренней структуры


**Тестирование белого ящика**(с полным знанием внутренней реализации)
- Цель: проверка логики, путей выполнения
- Техники: покрытие кода, мутационное тестирование
- Выполняют: Разработчики тестировщики с доступом к коду

**Тестирование черного ящика** (без знания реализации)
- Цель: Проверка соответствия требованиям
- Техники: эквивалентное разбиение, анализ граничных значений
- Выполняют: тестировщики, аналитики
- 
**Тестирование серого ящика**
- Цель: комбинированный подход
- тестирование API с знанием схемы данных


## По времени выполнения

**Дымовое тестирование**
	Быстрая проверка основных функций после сборки

**Регрессионное**
	Проверка что новое не сломало ничего старого

**Санитарное** 
	Углубленное тестирование после исправления дефектов

**Тестирование новой функциональности**
	фокус на недавно добавленных возможностях

## По принципу поведения

**Позитивное тестирование** Проверка с корректными данными

**Негативное тестирование** Проверка обработки некорректных данных и ошибок


## Специальные виды тестирования
**Конфигурационное тестирование**
	 Проверка работы в разных конфигурациях

**Локализационное тестирование**
	Проверка адаптации под разные языки и регионы

**Доступности (Accessibility)**
	Проверка доступности для людей с ограниченными возможностями

**Миграционное тестирование**
	Проверка переноса данных и настроек


## По степени изолированности
**Изолированное**
	Тестирование компонентов по отдельности

**Совместное тестирование**
	Тестирование компонентов в сборке
